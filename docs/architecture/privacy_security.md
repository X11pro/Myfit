# Privacidad y seguridad

## Principios

1. Pedir solo permisos necesarios.
2. Explicar por que se usa cada dato.
3. Permitir borrar y exportar datos.
4. No usar datos de salud para publicidad.
5. No enviar datos a IA sin consentimiento claro.

## Controles minimos

- `Row Level Security` en Supabase.
- Tokens en `flutter_secure_storage`.
- Cifrado en transito y en reposo donde aplique.
- Separacion de datos `Strava` si se integran.

## Limites del producto

- No diagnosticar.
- No prometer precision medica.
- No recomendar sustancias de alto riesgo.
- No almacenar rutas GPS en el MVP salvo necesidad real.
